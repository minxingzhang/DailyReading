TombRaider: Entering the Vault of History to Jailbreak Large Language Models
Junchen Ding, Jiahao Zhang, Yi Liu et al. 2025-01-27 EMNLP
核心贡献
TombRaider利用双智能体系统挖掘LLM的历史知识来生成对抗性提示，在无防御模型上实现近100%的攻击成功率。
TombRaider jailbreaks LLMs by using a two-agent system that exploits models' historical knowledge to craft adversarial prompts, achieving nearly 100% attack success rates on undefended models.
重要性与意义 / Significance
TombRaider揭示了LLM历史知识检索这一此前被忽视的攻击面，难以在不损害模型能力的情况下修补。该研究为红队社区提供了新的威胁模型，对未来防御基准测试具有重要参考价值。
TombRaider reveals a previously underexplored attack surface — LLMs' built-in historical knowledge retrieval — which is difficult to patch without degrading model utility. The near-100% ASR on bare models and 55.4%+ ASR against active defenses signals an urgent gap in current LLM safety alignment. This work provides the red-teaming community with a concrete new threat model to benchmark future defenses against.
科研问题 / Research Question
LLM存储和检索历史知识的能力能否被系统性地利用来绕过安全过滤器？当前的安全对齐主要针对有害输出过滤，而忽视了历史知识被重新包装为合法请求的风险。
Can the historical knowledge stored and retrieved by LLMs be systematically exploited to bypass their safety filters? This matters because safety alignment in current LLMs focuses on filtering harmful outputs but may overlook how historical or contextual knowledge can be weaponized to reframe harmful requests as legitimate ones.
先前工作为何不够 / Why Prior Work Falls Short
现有越狱方法（如DAN角色扮演、GCG对抗后缀攻击）要么计算成本高，要么对更新后的安全过滤器脆弱。这些方法均未系统利用模型自身的历史知识作为攻击向量。
Existing jailbreak methods — such as prompt injection, role-playing (e.g., DAN), and optimization-based adversarial suffix attacks (e.g., GCG) — either require heavy computational resources or are fragile against updated safety filters. Template-based approaches rely on static prompt patterns that are increasingly detectable by modern guardrails. None of these methods systematically leverage the model's own internal historical knowledge as an attack vector.
解决方案 / Solution
TombRaider采用双智能体流程：检查员智能体从目标LLM中提取与有害主题相关的历史事实，攻击者智能体将这些历史背景融入对抗性提示，将有害请求伪装成历史探究。该方法无需梯度访问或微调目标模型，具有较强的模型通用性。
TombRaider introduces a two-agent pipeline: an "inspector agent" that queries the target LLM to extract relevant historical facts, events, or figures related to a harmful topic, and an "attacker agent" that weaves this historical context into adversarial prompts to disguise harmful requests as historical inquiry. By grounding malicious requests in real historical knowledge, the attack reframes dangerous content as factual/educational discussion. This approach is model-agnostic and requires no gradient access or fine-tuning of the target model.
主要结论 / Key Results
TombRaider在六个无防御LLM上实现近100%的攻击成功率，超越所有基线方法。在主动防御机制下，ASR仍保持在55.4%以上，表明该方法对当前护栏技术具有较强的鲁棒性。
TombRaider achieved nearly 100% Attack Success Rate (ASR) on six undefended popular LLMs, outperforming all evaluated state-of-the-art jailbreak baselines. Against active defense mechanisms, TombRaider maintained an ASR exceeding 55.4%, demonstrating resilience to current guardrail techniques. These results held consistently across all six tested models, suggesting broad applicability of the historical knowledge exploitation vector.
值得关注的讨论点 / Discussion Points
  • 伦理双刃剑：发布近100% ASR的完整攻击方法，在防御措施广泛部署前存在即时滥用风险。
  • 防御缺口：55.4%的防御穿透率对生产系统仍极为危险，论文未提出具体防御对策，是重要的开放性问题。
  • 泛化局限：论文测试了六个模型，但未明确说明是否包含GPT-4o、Claude 3.5等闭源前沿模型，实际影响范围存疑。
  • Ethical dual-use tension: While the paper is framed as red-teaming research, publishing a near-100% ASR attack technique with full methodology creates immediate misuse risk before defenses are widely deployed.
  • Defense gap: A 55.4% ASR against existing defenses is still alarmingly high for production systems — the paper does not propose concrete countermeasures, leaving a critical open problem for the community.
  • Generalizability limits: The evaluation covers six models, but the paper does not specify whether closed-source models (e.g., GPT-4o, Claude 3.5) with more opaque safety layers were tested, leaving open questions about real-world impact on frontier models.

Invisible Prompts, Visible Threats: Malicious Font Injection in External Resources for Large Language Models
Junjie Xiong, Changjia Zhu, Shuhang Lin et al. 2025-05-22 EMNLP
核心贡献
攻击者可通过重映射字体字形表，向网页注入隐藏对抗性指令，从而绕过LLM安全机制。
This paper demonstrates that attackers can inject hidden adversarial instructions into webpages by remapping font code-to-glyph tables, bypassing LLM safety filters during web-augmented or MCP-integrated inference.
重要性与意义 / Significance
随着LLM广泛集成网络搜索与MCP工具协议，字体注入攻击面正快速扩大，威胁现实部署系统。本文是首批系统研究字体型间接提示注入的工作之一，为安全研究者提供了具体的威胁模型。
As LLMs are increasingly deployed with real-time web search and tool-use protocols like MCP, this attack surface is growing rapidly and affects production systems today. The paper provides one of the first systematic studies of font-based indirect prompt injection, offering a concrete threat model for practitioners and security researchers. Its findings call for immediate attention to input sanitization and rendering-layer defenses in LLM pipelines.
科研问题 / Research Question
攻击者能否通过操控字体字形映射，将恶意指令隐藏于网页内容中，使LLM执行隐藏文本而用户毫不知情？这一问题对具备网络检索或MCP工具访问能力的LLM部署尤为关键。
Can attackers hide malicious instructions inside external web content by manipulating font glyph mappings, such that LLMs read and execute the hidden text while human users see only benign content? This matters because LLMs with web retrieval or MCP tool access process raw HTML/font data that users never directly inspect, creating an asymmetric visibility gap ripe for exploitation.
先前工作为何不够 / Why Prior Work Falls Short
现有提示注入研究多针对直接用户输入或可见对抗文本，防御手段集中于输出过滤与系统提示加固，对字体编码层的安全性研究几乎空白。
Prior prompt injection research mostly targets direct user inputs or visible adversarial text embedded in retrieved documents. Existing defenses focus on output filtering and system-prompt hardening, leaving the rendering layer—where font encoding is interpreted—largely unexamined. No prior work systematically studied code-to-glyph manipulation as a steganographic vector for indirect prompt injection.
解决方案 / Solution
作者通过重映射Unicode码位与字形的对应关系，构造恶意字体，使网页在浏览器中显示为无害内容，但LLM分词器解码后得到隐藏的攻击指令。实验针对"恶意内容中继"与"敏感数据泄露"两种场景，覆盖多款LLM与多种提示设计。
The authors craft malicious fonts by remapping Unicode code points to different glyphs, so that a string visually reads as innocuous text to a human browser but is decoded by the LLM's tokenizer as a hidden adversarial instruction. They embed these fonts in external webpages and evaluate two attack scenarios: "malicious content relay" (LLM reproduces attacker-chosen harmful content) and "sensitive data leakage" (LLM exfiltrates private context via MCP-enabled tools). Experiments span multiple LLMs and prompt designs to measure attack success rates under varying data sensitivity levels.
主要结论 / Key Results
字体注入攻击在多款主流LLM上成功绕过安全机制，攻击成功率随数据敏感性与提示设计而变化。经由MCP工具的敏感数据泄露场景尤为有效，实验证明该攻击具有实际可行性。
Malicious font injection successfully bypasses LLM safety mechanisms across tested models, with attack success rates varying based on the sensitivity of the targeted data and the specificity of the injected prompt design. The "sensitive data leakage" scenario via MCP tools proves particularly effective, as tool-use pipelines apply less scrutiny to retrieved external content. Exact aggregate success rates are not uniformly reported, but results confirm the attack is practically viable across multiple frontier LLMs.
值得关注的讨论点 / Discussion Points
  • 攻击面边界不清：攻击依赖LLM处理原始字体字节而非渲染字形，不同检索管道的实际漏洞范围仍需进一步厘清。
  • 防御方案缺失：论文识别威胁但未提出或评测具体缓解措施（如字形归一化预处理、HTML字体剥离），实践指导有限。
  • 泛化潜力：编码层感知差异这一核心思路可能延伸至同形字替换、零宽字符等其他隐写通道，值得进一步研究。
  • Scope of font attack surface: The attack relies on the LLM processing raw font-encoded bytes rather than rendered glyphs, which is an architectural assumption that may not hold uniformly across all retrieval pipelines—future work should map exactly which ingestion architectures are vulnerable.
  • Defenses are underspecified: The paper identifies the threat but does not propose or benchmark concrete mitigations (e.g., glyph-normalizing preprocessors, font-stripping in HTML parsers), leaving practitioners without actionable remediation guidance.
  • Generalization beyond fonts: The core idea—exploiting encoding-layer mismatches between human perception and LLM tokenization—likely generalizes to other steganographic channels (e.g., homoglyph Unicode substitution, invisible zero-width characters), suggesting a broader class of perception-gap attacks worth investigating.

G-Safeguard: A Topology-Guided Security Lens and Treatment on LLM-based Multi-agent Systems
Shilong Wang, Gui-Min Zhang, Miao Yu et al. 2025-02-16 ACL
核心贡献
G-Safeguard利用图神经网络检测多智能体对话图中的异常并进行拓扑干预，可恢复因提示注入攻击损失的40%以上性能。
G-Safeguard uses graph neural networks on multi-agent utterance graphs to detect and remediate adversarial attacks in LLM-based multi-agent systems, recovering over 40% of performance lost to prompt injection.
重要性与意义 / Significance
随着LLM多智能体系统被部署于高风险领域，G-Safeguard提供了一个可扩展、与骨干模型无关的图论安全框架，为多智能体安全研究树立了方法论基准。
As LLM-based multi-agent systems are deployed in high-stakes domains, the lack of robust defenses against adversarial manipulation is a critical gap. G-Safeguard provides a principled, graph-theoretic security framework that is both backbone-agnostic and scalable, making it directly applicable to real-world MAS deployments. This work sets a concrete methodological baseline for the emerging field of multi-agent security.
科研问题 / Research Question
LLM多智能体系统在自主决策中应用广泛，但面对提示注入和虚假信息传播等对抗攻击时极为脆弱。现有防御方法主要针对单智能体设计，无法利用MAS中的智能体交互拓扑结构，本文探索如何借助拓扑信息检测并消除攻击。
LLM-based multi-agent systems (MAS) are increasingly used for autonomous decision-making and collaborative problem-solving, yet they remain highly vulnerable to adversarial attacks such as prompt injection and misinformation propagation. Existing defenses are largely designed for single-agent settings and fail to account for the complex inter-agent communication topology inherent in MAS. This paper asks: how can the structural topology of agent interactions be leveraged to detect and neutralize attacks in MAS?
先前工作为何不够 / Why Prior Work Falls Short
现有LLM对抗防御（输入过滤、提示加固等）仅针对单智能体流程，忽视了多智能体间消息传播的图结构特性。图异常检测虽有研究，但尚未被应用于MAS动态对话图，也无统一框架同时实现检测与拓扑干预。
Prior defenses against LLM adversarial attacks—such as input filtering, output verification, and prompt hardening—are designed for single-agent pipelines and do not model inter-agent message propagation. Graph-based anomaly detection has been studied in network security but has not been applied to the dynamic utterance graphs formed in MAS communication. No existing work provides a unified framework that simultaneously handles detection and topological intervention across diverse MAS architectures.
解决方案 / Solution
G-Safeguard将智能体通信建模为对话图，用GNN检测异常节点或边（如注入提示或被篡改消息），再通过拓扑干预模块隔离或重路由被攻击的通信路径。该框架模块化设计，可与主流MAS平台及任意LLM骨干无缝集成。
G-Safeguard models the communication among agents as a multi-agent utterance graph, where nodes represent agents and edges encode message exchanges. A graph neural network (GNN) is trained to detect anomalous nodes or edges that signal adversarial activity, such as injected prompts or corrupted messages. Upon detection, a topological intervention module surgically isolates or re-routes compromised communication paths to remediate the attack. The framework is modular, enabling seamless integration with mainstream MAS platforms and arbitrary LLM backbones.
主要结论 / Key Results
G-Safeguard在提示注入攻击下可恢复超过40%的任务性能损失，并在多种LLM骨干和大规模MAS配置上均表现出良好的泛化能力与可扩展性，集成开销小。
G-Safeguard recovers over 40% of task performance degraded by prompt injection attacks, demonstrating strong effectiveness. It generalizes across multiple LLM backbones (e.g., GPT series, open-source models) and scales to large MAS configurations. The framework also integrates with mainstream MAS architectures without significant overhead, confirming its practical adaptability.
值得关注的讨论点 / Discussion Points
  • 拓扑作为安全信号：将智能体通信建模为图是核心创新——对抗扰动必然扭曲对话图结构，GNN异常检测因此具有天然优势。
  • 可扩展性的代价：动态对话图上的实时GNN推理开销尚需深入分析，尤其在对延迟敏感的应用场景中。
  • 泛化能力待验证：40%恢复率针对提示注入表现突出，但对虚假信息传播、后门智能体等其他攻击类型的量化结果仍需补充，以建立更全面的鲁棒性保证。
  • Topology as a security signal: Modeling agent communications as a graph is a key insight — adversarial perturbations inevitably distort the utterance graph structure, making GNN-based anomaly detection a natural and powerful fit for this setting.
  • Scalability and integration trade-offs: While the paper claims scalability to large-scale MAS, the computational cost of running GNN inference on dynamically evolving utterance graphs at inference time deserves deeper analysis, especially for latency-sensitive applications.
  • Generalization beyond prompt injection: The 40% recovery metric is compelling for prompt injection, but the paper should more thoroughly quantify performance against other attack types (e.g., misinformation propagation, backdoor agents) to establish broader robustness guarantees.

MASFactory: A Graph-centric Framework for Orchestrating LLM-Based Multi-Agent Systems with Vibe Graphing
Yang Liu, Jinxuan Cai, Yishen Li et al. 2026-03-06 ACL
核心贡献
MASFactory提出以图为核心的框架，通过"Vibe Graphing"将自然语言意图编译为可执行的多智能体工作流图，大幅降低手动实现成本。
MASFactory introduces a graph-centric framework with "Vibe Graphing" that compiles natural-language descriptions into executable multi-agent workflow graphs, dramatically reducing manual implementation effort.
重要性与意义 / Significance
MASFactory解决了多智能体系统部署中的关键瓶颈——复杂工作流的高昂手动配置成本。其自然语言驱动的图构建方式降低了研究者与从业者的使用门槛，可复用组件与可插拔上下文集成进一步提升了实用性。
MASFactory tackles a critical bottleneck in deploying LLM-based multi-agent systems: the high manual overhead of wiring complex workflows. By enabling natural-language-driven graph construction with human-in-the-loop editing, it lowers the barrier for both researchers and practitioners to build and iterate on sophisticated MAS pipelines. The reusable components and pluggable context integration further make it practically deployable across heterogeneous settings.
科研问题 / Research Question
如何让开发者高效构建、复用和集成复杂的多智能体工作流，而无需大量手动编码？当前框架在实现、调试和扩展图结构工作流时工程代价高昂，亟需解决。
How can developers efficiently construct, reuse, and integrate complex LLM-based multi-agent workflows without heavy manual coding? This matters because MAS workflows naturally map to directed computation graphs, yet current frameworks require significant engineering effort to implement, debug, and extend them.
先前工作为何不够 / Why Prior Work Falls Short
现有框架（如LangGraph、AutoGen）需开发者手动以代码定义智能体角色、边和消息传递逻辑，复用性差且易出错。异构外部上下文（如RAG、API、记忆模块）的集成支持有限，图工作流的可视化与运行时追踪能力也较为薄弱。
Existing MAS frameworks (e.g., LangGraph, AutoGen) require developers to manually define agent roles, edges, and message-passing logic in code, which is error-prone and hard to reuse. They also offer limited support for integrating heterogeneous external context sources (e.g., RAG, APIs, memory stores) in a modular, plug-and-play manner. Visualization and runtime tracing capabilities for debugging graph-structured workflows are largely underdeveloped.
解决方案 / Solution
MASFactory将多智能体工作流建模为有向计算图，节点为智能体或子工作流，边编码依赖与消息传递。核心创新"Vibe Graphing"以人在回路的方式将自然语言意图转化为可编辑的工作流规范，再编译为可执行图。框架还提供可复用组件库、可插拔上下文集成层及支持拓扑预览与运行时追踪的可视化工具。
MASFactory models MAS workflows as directed computation graphs where nodes represent agents or sub-workflows and edges encode dependencies and message passing. Its core innovation, Vibe Graphing, is a human-in-the-loop pipeline that translates natural-language intent into an editable workflow specification (an intermediate representation), which is then compiled into an executable graph. The framework also provides a library of reusable agent/tool components and a pluggable context integration layer supporting diverse external sources. A built-in visualizer supports topology preview, runtime execution tracing, and mid-run human intervention.
主要结论 / Key Results
本文以系统框架为主，未提供大规模定量基准测试结果。通过工作流案例展示了Vibe Graphing可成功将自然语言规范编译为可执行图，并减少手动工作量，但未给出与LangGraph等框架的具体性能对比数据。
The paper is primarily a system/framework paper and does not report large-scale quantitative benchmarks. It demonstrates MASFactory's capabilities through system design descriptions and workflow case studies, showing that Vibe Graphing can successfully compile natural-language specifications into executable graphs with reduced manual effort. Specific runtime performance figures or comparative benchmark numbers against LangGraph or AutoGen are not provided in the abstract.
值得关注的讨论点 / Discussion Points
  • Vibe Graphing依赖LLM解析自然语言意图，模糊输入可能生成错误图结构，论文未量化编译失败率或错误恢复机制，可靠性存疑。
  • 人在回路设计有助于确保正确性，但在追求高度自动化的流水线场景中可能成为瓶颈，自动化与人工监督之间的权衡需进一步探讨。
  • 缺乏与LangGraph、AutoGen在标准化任务（如GAIA、AgentBench）上的严格对比实验，难以评估框架抽象是否引入额外开销或切实提升开发效率。
  • Vibe Graphing's reliance on LLMs to parse natural-language intent into workflow specs raises reliability concerns — ambiguous or underspecified inputs may generate incorrect graph structures, and the paper does not quantify compilation failure rates or error recovery mechanisms.
  • The human-in-the-loop design is a strength for correctness but may become a bottleneck in automated pipelines where minimal human intervention is desired; the tradeoff between automation and oversight needs further exploration.
  • Without rigorous benchmarks comparing MASFactory against LangGraph or AutoGen on standardized tasks (e.g., GAIA, AgentBench), it is difficult to assess whether the framework's abstractions introduce overhead or genuinely improve developer productivity at scale.

COLD-Steer: Steering Large Language Models via In-Context One-step Learning Dynamics
Kartik Sharma, Rakshit S. Trivedi 2026-03-06 ICLR
核心贡献
COLD-Steer是一种免训练激活引导框架，通过近似上下文样本梯度下降的表示变化，解决了LLM行为控制中样本效率与信号质量之间的权衡难题。
COLD-Steer is a training-free activation steering framework that approximates the representational effect of gradient descent on in-context examples, resolving the sample-efficiency vs. signal-quality trade-off in LLM behavioral control.
重要性与意义 / Significance
COLD-Steer无需参数更新或大量标注数据即可实现细粒度推理时控制，降低了受限资源场景下部署引导模型的门槛。其免训练特性使其可直接应用于任何预训练LLM，对无法承担反复微调的实践者具有重要价值。
COLD-Steer offers a practical path to fine-grained inference-time control of LLMs without any parameter updates or large labeled datasets, lowering the barrier for deploying steered models in resource-constrained settings. Its training-free nature makes it immediately applicable to any pre-trained LLM, which is significant for practitioners who cannot afford repeated fine-tuning cycles.
科研问题 / Research Question
如何仅使用少量标注样本在推理时引导LLM行为，而无需重新训练或构建大规模对比数据集？这对于在部署中将LLM输出对齐到特定任务或安全约束至关重要。
How can we steer LLM behavior at inference time using only a small number of labeled examples, without retraining or expensive contrastive dataset construction? This matters because reliable, lightweight behavioral control is essential for aligning LLM outputs to specific tasks or safety constraints in deployment.
先前工作为何不够 / Why Prior Work Falls Short
ActAdd、CAA等样本高效方法从少量配对数据中提取引导向量，但信号捕获能力有限；而基于探针或表示微调的方法虽信号质量高，却需数百至数千标注样本，部署成本高昂。两类方法均无法有效兼顾数据效率与高保真行为控制。
Sample-efficient methods like Activation Addition (ActAdd) and Contrastive Activation Addition (CAA) extract steering vectors from small paired datasets but fail to capture rich directional signals, limiting steering quality. Methods with stronger signal extraction, such as learned probes or representation fine-tuning, require hundreds to thousands of labeled examples, making them costly to deploy. Neither category effectively bridges the gap between data efficiency and high-fidelity behavioral control.
解决方案 / Solution
COLD-Steer近似上下文样本经一步梯度下降后的激活空间变化，无需更新任何模型参数。单位核近似方法利用对激活值的归一化梯度直接更新激活；有限差分近似方法仅需两次前向传播即可估计激活空间的方向偏移，计算开销极低。两种方法协同工作，使COLD-Steer能从少量样本中提取更丰富的引导信号，同时保持免训练特性。

The first approach, unit kernel approximation, directly updates activations using normalized gradients with respect to those activations across the example set. The second approach, finite-difference approximation, estimates the directional shift in activation space using only two forward passes, making it computationally lightweight regardless of the number of examples. Together, these two methods allow COLD-Steer to extract richer steering signals from few examples while remaining entirely training-free.

值得关注的讨论点 / Discussion Points
  • 近似保真度：两种方法均为近似而非精确的梯度下降动态，当损失曲面高度非线性或上下文样本存在噪声时，引导信号质量可能下降。
