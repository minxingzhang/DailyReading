Fun-tuning: Characterizing the Vulnerability of Proprietary LLMs to Optimization-Based Prompt Injection Attacks via the Fine-Tuning Interface
Andrey Labunets, Nishit V. Pandya, Ashish Hooda et al. 2025-01-16 IEEE Symposium on Security and Privacy 评分 9.8
核心贡献
Fun-tuning利用商业LLM微调API返回的损失信号，实现黑盒对抗性提示注入攻击，攻击成功率达65%–82%。
Fun-tuning exploits the loss signals returned by proprietary LLM fine-tuning APIs (e.g., Gemini) to perform black-box optimization-based prompt injection attacks, achieving 65–82% attack success rates.

科研问题 / Research Question
封闭权重的LLM通常被认为可以抵御基于梯度的攻击。本文探讨攻击者能否滥用微调接口提取足够的优化信号，对Gemini等专有模型发动对抗性提示注入攻击。
Closed-weight LLMs are generally considered safe from gradient-based adversarial attacks because their weights are inaccessible. This paper asks: can an attacker abuse the fine-tuning interface—a standard developer utility—to extract enough optimization signal to craft adversarial prompt injections against proprietary models like Gemini?
先前工作为何不够 / Why Prior Work Falls Short
GCG等现有优化攻击需直接访问模型权重和梯度，无法用于封闭权重模型。基于迁移的黑盒攻击依赖开源替代模型，但对专有架构迁移效果差。此前无研究系统性地将微调API作为对抗性提示优化的攻击面。
Existing optimization-based attacks like GCG require direct access to model weights and gradients, making them inapplicable to closed-weight, API-only LLMs. Transfer-based black-box attacks rely on open-source surrogate models, but suffer from poor transferability to proprietary architectures. No prior work systematically exploited the fine-tuning API as an attack surface for adversarial prompt optimization.
解决方案 / Solution
作者提出"Fun-tuning"方法，将Gemini微调API返回的训练损失值作为代理优化信号，并使用贪心token替换搜索算法迭代生成对抗性提示后缀。该攻击无需模型权重或替代模型，仅需普通开发者API访问权限，并在PurpleLlama基准上对Gemini系列模型进行评估。
The authors introduce "Fun-tuning," a method that treats the training loss values returned by a vendor's fine-tuning API (specifically Gemini's) as a proxy optimization signal. They use a greedy token-substitution search algorithm—similar in spirit to GCG but operating purely on API-returned loss values—to iteratively craft adversarial prompt suffixes. The attack requires no model weights, no surrogate model, and only standard API access available to any developer. Evaluation is performed on the PurpleLlama prompt injection benchmark against the Gemini family of models.
主要结论 / Key Results
Fun-tuning在PurpleLlama基准上对Gemini系列模型的攻击成功率达65%–82%，显著优于朴素黑盒基线。微调API返回的损失信号足以引导离散贪心搜索，且该漏洞在多个Gemini模型变体上普遍存在，具有系统性。
Fun-tuning achieves prompt injection attack success rates of 65%–82% across Google's Gemini family of models on the PurpleLlama benchmark, substantially outperforming naive black-box baselines. The fine-tuning API's loss-like signal proves sufficient to guide discrete greedy search despite offering far less information than direct gradient access. The results hold across multiple Gemini model variants, suggesting the vulnerability is systemic rather than model-specific.
值得关注的讨论点 / Discussion Points
  • 实用与安全的权衡：微调接口是厂商提供开发者功能的设计选择，但该研究表明其天然泄露足够的优化信号，这一矛盾难以通过简单删减功能解决。
  • 规模与成本：基于远程API的贪心token搜索需大量顺序调用，攻击的实际查询成本与速率限制下的鲁棒性有待量化，未来可研究损失值扰动或查询节流等防御措施。
  • 更广泛适用性：尽管本文仅在Gemini上验证，任何返回损失值的微调API（如OpenAI、Cohere）都可能存在类似漏洞，相关厂商应审查其训练指标是否暴露等价信号。
  • Utility-security tradeoff: The fine-tuning interface is a deliberate design choice by vendors to provide developer utility, yet this paper shows it inherently leaks enough optimization signal to enable adversarial attacks—a tension that cannot be resolved simply by removing features.
  • Scalability and cost: Greedy token-search over a remote API involves many sequential API calls, raising questions about the practical cost and rate-limit resilience of the attack. Future work should quantify query budgets and explore defenses such as loss-value perturbation or query throttling.
  • Broader applicability: While demonstrated on Gemini, any LLM vendor offering a fine-tuning API that returns per-sample or aggregate loss values is potentially vulnerable. OpenAI, Cohere, and others with similar interfaces should audit whether their returned training metrics expose comparable signals.

GAP用互联图结构替代树状越狱框架，实现跨攻击路径的知识共享，攻击成功率超96%，查询成本降低62.7%。

科研问题 / Research Question
随着LLM大规模部署，绕过安全护栏的越狱攻击带来严重风险。本文探讨如何更高效地生成隐蔽、高成功率的越狱提示，以压力测试并强化LLM内容审核系统。
As LLMs are deployed at scale, adversarial jailbreak attacks that bypass safety guardrails pose serious risks. The paper asks: how can we generate stealthy, high-success-rate jailbreak prompts more efficiently to stress-test and ultimately strengthen LLM content moderation systems?
先前工作为何不够 / Why Prior Work Falls Short
现有方法（如TAP、PAIR）采用树状或单路径搜索结构，导致一条分支发现的有效攻击策略无法被其他分支复用，造成查询浪费，对防御严密的闭源LLM攻击成功率有限。
Existing jailbreak methods like TAP and PAIR rely on tree-based or single-path search structures, which prevent successful attack strategies discovered in one branch from informing other branches. This siloed design wastes queries and limits attack success rates, especially against well-defended closed-source LLMs.
解决方案 / Solution
GAP将越狱搜索过程建模为有向图而非树，使成功的提示模式可同时传播至多条攻击路径。剪枝机制提前淘汰低潜力节点，将查询预算集中于最有效策略。GAP-Auto实现种子提示自动生成，GAP-VLM将方法扩展至多模态（视觉-语言）模型，使其成为适用于开源与闭源LLM的通用红队测试工具。
GAP (Graph of Attacks with Pruning) models the jailbreak search process as a directed graph rather than a tree, allowing successful prompt patterns to propagate across multiple attack paths simultaneously. A pruning mechanism eliminates low-promise nodes early, concentrating the query budget on the most effective strategies. The framework also introduces GAP-Auto, which automates seed prompt generation without manual input, and GAP-VLM, which extends the approach to multimodal (vision-language) models. Together, these components make GAP a general-purpose red-teaming toolkit applicable to both open- and closed-source LLMs.
主要结论 / Key Results
GAP对开源与闭源LLM的攻击成功率均超96%，较最先进基线提升20.8%，同时查询次数减少62.7%。GAP生成的对抗提示还可用于训练内容审核分类器，提升其对未知越狱攻击的鲁棒性。
GAP achieves attack success rates exceeding 96% against both open- and closed-source LLMs, representing a 20.8% improvement over state-of-the-art baselines. Simultaneously, it reduces the number of queries required by 62.7%, making it substantially more cost-efficient. GAP-generated adversarial prompts also prove effective as training data for content moderation classifiers, improving their robustness against unseen jailbreak attacks.
值得关注的讨论点 / Discussion Points
  • 图结构优于树结构是核心洞见：跨分支共享信息避免了树状方法的冗余探索，该设计原则可推广至其他LLM红队测试或提示优化任务。
  • 双重用途定位处理得当：GAP明确服务于提升内容审核而非单纯攻击，将生成的对抗提示用于强化分类器，展示了超越基准数字的实际安全价值。
  • 通过GAP-VLM扩展至多模态模型具有重要意义，但论文对VLM的评估范围相比纯文本实验明显有限，跨模态攻击的泛化能力仍有待深入验证。
  • Graph over tree is the key insight: by allowing nodes to share information across branches, GAP avoids redundant exploration that plagues tree-based methods — this design principle could generalize to other LLM red-teaming or prompt optimization tasks.
  • The dual-use framing is well-handled: GAP is explicitly positioned as a tool for improving content moderation, not just attacking, and the use of GAP-generated prompts to harden classifiers demonstrates a concrete safety benefit beyond benchmark numbers.
  • Scalability to multimodal models via GAP-VLM is significant, as vision-language models introduce new attack surfaces; however, the paper's evaluation scope for VLMs appears limited compared to text-only experiments, leaving open questions about cross-modal attack generalization.

The authors construct CyberLLMInstruct, a dataset of 54,928 pseudo-malicious instruction-response pairs sourced from CTF challenges, CVE databases, academic papers, and industry reports, covering malware analysis, phishing simulations, and zero-day vulnerabilities. Seven open-source LLMs (including Llama 3.1 8B, Mistral, Phi) are fine-tuned on this dataset and evaluated on both cybersecurity task accuracy (CyberMetric benchmark) and safety resilience against multiple attack vectors including prompt injection and jailbreaks. The study quantifies the trade-off between domain performance gains and safety score degradation, providing a structured framework for diagnosing alignment failures in adversarial fine-tuning scenarios.
主要结论 / Key Results

Fine-tuning on CyberLLMInstruct achieves up to 92.50% accuracy on the CyberMetric cybersecurity benchmark, confirming significant domain performance gains. However, safety resilience collapses dramatically: Llama 3.1 8B's security score against prompt injection drops from 0.95 to 0.15 after fine-tuning. This degradation is consistent across all seven tested models and attack vectors, indicating a systemic rather than model-specific vulnerability.

科研问题 / Research Question
具身LLM在控制物理设备时面临新型安全威胁：通过语言层安全检测的指令仍可触发危险的真实世界动作。随着AI智能体被部署于机器人、自动驾驶等领域，这种语言安全与物理结果之间的错位风险日益严峻。
Embodied LLMs that control physical agents face a novel security threat: instructions that pass language-level safety checks can still trigger harmful real-world actions. This misalignment between linguistic safety and physical outcomes poses serious risks as AI agents are deployed in robotics, autonomous vehicles, and smart environments.
先前工作为何不够 / Why Prior Work Falls Short
现有LLM越狱研究主要针对文本输出，通过提示工程或黑盒迭代攻击实现，难以迁移至具身场景。此前工作普遍忽视语义到动作的鸿沟：语言上安全的响应在物理执行时可能造成灾难性后果。
Existing LLM jailbreak research focuses primarily on eliciting harmful text outputs through prompt engineering or iterative black-box attacks, which do not transfer well to the embodied setting where harm manifests through physical actions rather than language. Prior work largely overlooks the semantic-to-action gap: a response can be linguistically safe yet physically catastrophic when actuated in the real world.
解决方案 / Solution
Blindfold采用对抗性代理规划策略：通过攻破本地代理LLM生成语义安全但物理有害的动作序列，而非直接攻击黑盒目标系统。关键恶意动作通过注入技术隐藏于行动计划中，规避安全监测，同时保持对不同具身LLM后端的高度可迁移性。
Blindfold introduces an Adversarial Proxy Planning strategy: instead of directly attacking a black-box embodied LLM, it compromises a local surrogate LLM to craft action sequences that appear semantically safe but produce harmful physical effects upon execution. Malicious key actions are concealed within the action plan through injection techniques, making them hard to detect by safety monitors.
